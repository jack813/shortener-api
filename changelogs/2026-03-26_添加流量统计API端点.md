## 修改原因

Dashboard 页面的 TrafficCard 组件无法显示，原因是调用的 `/api/traffic` 端点在 shortener-api 项目中不存在。流量相关的 API 代码原本在 shortener-link 项目中编写，但实际 API 请求被代理到 api.0x1.in（由 shortener-api 服务），导致流量统计数据无法获取。

## 修改步骤

1. 新增 `src/lib/traffic-quota.ts`，实现流量配额管理功能
   - `getCurrentMonth()` - 获取当前月份
   - `checkTrafficQuota()` - 检查用户流量配额
   - `getTrafficUsage()` - 获取用户流量使用量
   - `getLinkTrafficUsage()` - 获取单个链接流量
   - `getResetTime()` - 获取配额重置时间
   - `incrementTraffic()` - 增加流量计数

2. 新增 `src/app/api/traffic/route.ts`，提供流量概览 API
   - `GET /api/traffic` - 返回用户本月流量使用情况

3. 新增 `src/app/api/traffic/links/route.ts`，提供链接级流量 API
   - `GET /api/traffic/links?page=1&limit=50` - 返回分页的链接流量数据

4. 更新 `src/lib/config/pricing.ts`，添加流量限制配置
   - Free 套餐：500 次/月
   - Pro 套餐：20,000 次/月

## 修改结果

- Dashboard 的 TrafficCard 组件现在可以正常显示流量数据
- 流量 API 端点已部署到 Cloudflare Pages
- `/api/traffic` 返回流量概览（used, limit, remaining, plan, resetAt）
- `/api/traffic/links` 返回分页的链接流量数据

## 备注

- 流量数据存储在 KV 中，TTL 为 33 天
- 流量计数采用 read-modify-write 方式，高并发下可能有少量丢失，但适合配额跟踪场景
