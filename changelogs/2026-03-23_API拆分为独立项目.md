## 修改原因

原有的 shortener-link 项目同时承担了三种职责：
1. 短链接重定向服务 (0x1.in)
2. API 服务 (shortener.0x1.in/api/*)
3. MCP 端点

为了实现高内聚、低耦合的架构，需要将 API 部分拆分为独立项目，实现：
- 职责分离：API 逻辑与重定向/页面服务解耦
- 独立部署：API 可独立扩展和更新
- 代码组织：Next.js 提供更好的路由结构和中间件支持

1. 创建 `shortener-api/` 目录
2. 初始化 Next.js 16 项目

### Phase 3: API 路由迁移

- `POST /api/create` - 创建短链接

- `GET/POST /api/keys` - API Key 管理
- `DELETE/PATCH /api/keys/[id]` - 删除/重命名 Key
- `GET /api/stats/[code]` - 统计数据
- `GET /api/lookup/[code]` - 查找链接
- `GET /api/me` - 当前用户
- `GET /api/quota` - 配额信息
- `GET /api/qr/[code]` - QR 码生成
- `POST /api/qr/logo` - Logo 上传
